# 体験フロー 確認セッション進捗

## 2026-07-13 追記: GPU 換装（RTX 5080）トラブルと対処

- 症状: TSDF メッシュが一切出ない（点群は出る）。`d3d12: CreateComputePipelineState failed [8007000E]`
- 切り分け: シェーダーキャッシュ全削除→改善せず / コンピュート再インポート→改善せず / **`-force-d3d11` 起動→完全復活（ゴーストで 257,640 tris）**
- 結論: **DX12 × RTX 5080（driver 591.86）の相性で TSDF 系コンピュートの PSO 作成が失敗**。他のコンピュート（点群再構成・占有・BT）は DX12 でも動くため「点群だけ見える」症状になる
- 対処: **プロジェクトを D3D11 固定**（PlayerSettings、コミット済み）。ドライバ/Unity 更新後に DX12 を再試行してもよい
- 併せて確定: DwellSphere の見た目一式（ICO球+黒芯+ビルボード時計リング 3cm 幅）はユーザー承認済み・コミット済み（b0e0d75）

## セッションで確定した仕様変更（すべて実装・コミット済み）

1. **rebase 常時 ON**（Dev/体験で座標系統一） — シーン値で `applyWorldRebase=true` + serial 順 N→L→EG→Z
2. **床 = y0** — `rebaseFloorY=-0.9`（SensorManager/SensorRecorder）で物理床が y=0 に。カメラは実測どおり床上 ~1m（N=0.90 / L=1.03 / Z=0.82 / EG=0.86）
3. **グリッドをワールド座標スナップ + 箱内のみ** — `FloorOrigin.fitToBoundingBox=true`。原点は必ず線の交点
4. **Fault アラートは太字 + 漢字**「カメラ（ID n）が異常です」（子供向けひらがなをやめた。API 名 KidsAlertText→FaultAlertText）
5. **QR はキャプション無し・仮表示** — 最終的に URL 表示は**別マシン**（IUrlPresenter の差し替えで対応予定、OscUrlPresenter 等）
6. **DwellSphere の見た目を全面変更**:
   - 球は**ICO球ワイヤー**（icoSubdivisions=1、120エッジ）で常時フル表示
   - 進捗 = 球の周りの**水平リング**が 0→一周で決定（円弧が閉じる旧仕様を廃止）
   - **裏面フェード**（DwellWire シェーダー `_BackFade=0.25`、カメラ毎に正しい奥側）
   - 半径 0.18 → **0.25m**（ユーザー実測チューニング）
